def parse_linestring(geometry):
    """
    :param geometry: a string in the format "LINESTRING(x1 y1, x2 y2, ..., xn yn)"
    return numpy array of shape (n, 2) with coordinates
    """
    start = geometry.find('(')
    end = geometry.find(')')
    substring = geometry[start+1:end]
    return np.array(
        [[float(f) for f in pair.split(' ')] for pair in substring.split(',')])

import math
def route_length(line): 
    """
    distance of a line defined by a sequence of n waypoints 
    :param line: is a numpy array of shape (n, 2) 
    :return: length of the line in km
    """
    earth_radius = 6371
    assert len(cities) >=2, 'at least two cities are requied'
    
    point1 = line[0]
    total = 0.0
    for point2 in line[1:]:
        x1, y1 = math.radians(point1[0]), math.radians(point1[1])
        x2, y2 = math.radians(point2[0]), math.radians(point2[1])
        total += math.asin(math.sqrt(
            math.sin((x2 - x1) / 2)**2 + 
            math.cos(y1) * math.cos(y2) * math.sin((y2 - y1) / 2)**2))
        point1 = point2
    return total * 2 * earth_radius

5. A box plot of total lengths of bottom 15 city's transit lines.¶

I wanted to see how a boxplot would graph against the data. I took the same code given for the bargraph and edited to see what other graph would work, and a boxplot was one of the few that worked. Instead of top 15 cities, I decided on top 25 cities, as the bottom 25 on this dataset wasn't as meaningful. I wanted to show the differences in length via quartiles on the boxplot. I tried different code to fill in the boxplot but I was unsuccessful, so I colored the lines. The 6-25 longest transit lines are closer in length than the top 5.¶
